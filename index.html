Regra da Cascata

O CSS funciona como uma “cascata”: quando há mais de uma regra para o mesmo elemento, o navegador decide qual usar considerando três fatores:

  • Origem da regra (CSS do autor, do navegador ou do usuário).
  • Especificidade do seletor (quanto mais específico, maior prioridade).
  • Ordem de aparição (em caso de empate, vale a última regra escrita).

Especificidade

Tag (ex.: p {}) → peso baixo

Classe/pseudo-classes (ex.: .box {}, :hover) → peso médio

ID (ex.: #menu {}) → peso alto

!important → maior peso de todos

!important sobrescreve quase tudo (mas é recomendável evitar).
